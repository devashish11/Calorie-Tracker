# Polish & Design Pass — 2026-07-20

## What was reviewed

After the AI frontend integration (tabs, text search, image upload), a full design review identified these gaps against the quality bar:

- **No meal categories** — all entries appeared in a flat undifferentiated list
- **Calorie progress bar lacked nuance** — only green/red, no amber warning zone
- **Macros had no visual** — numbers only, no sense of "how far" toward daily target
- **Header was minimal** — generic "Calorie Tracker" text, no brand identity
- **Font was system-default** — inconsistent rendering across platforms
- **FoodLog entries were text rows** — no visual hierarchy, hard to scan
- **Empty state was a text line** — no illustration or helpful context
- **Tab labels had no visual** — "Quick Add / Describe Meal / Photo" — functional but flat

## What was improved

### Meal categories
Added `meal_category` column to SQLite (`ALTER TABLE` with try/catch for existing DBs). `AddFoodPanel` gained a 4-pill category selector (Breakfast, Lunch, Dinner, Snack) with a smart time-of-day default (before 11am → Breakfast, 11–14 → Lunch, 14–20 → Dinner, after 20 → Snack). Both add callbacks inject the selected category. `FoodLog` groups entries by category with emoji section headers and per-section calorie totals.

### NutritionSummary redesign
- Large "consumed / goal" display with the number prominent
- Progress bar now three-state: emerald (< 75%), amber (75–99%), red (≥ 100%)
- Macro sub-bars added (protein blue, carbs amber, fat rose) with `goal / current` labels
- Daily macro targets: 150g protein, 250g carbs, 65g fat

### FoodLog redesign
- Sections per meal with emoji icon, category name, and total calories for that meal
- Each entry is a card (`bg-gray-50 rounded-xl`) with food name, serving size, time logged
- Macros shown as coloured values (blue/amber/rose) on larger screens
- Delete button is a trash SVG icon that fades in on row hover — less visual clutter
- Empty state has an illustration emoji, a headline, and a helpful instruction

### Header & brand
- App renamed to **NutriTrack** in both `layout.tsx` (page `<title>`) and `page.tsx`
- Top accent bar: 1px emerald → teal gradient across the full viewport width
- Header logo: `◉` circle symbol in emerald, bold tracking-tight type — no image dependency
- Date rendered in a slightly more compact position below the logo

### Inter font
- `next/font/google` loads Inter with the `latin` subset
- CSS variable `--font-inter` applied via `@theme` in `globals.css`
- Body class set to `font-sans`, resolving to Inter everywhere consistently

### CSS polish
- `scroll-behavior: smooth` on `<html>`
- Custom `::selection` colour: emerald-100 background, dark emerald text — reinforces brand in copy

## Verification

- Server compiled all changes without TypeScript errors
- API smoke test: POST with `meal_category: "Breakfast"` and `meal_category: "Lunch"` returned correct entries with the field persisted
- `GET /` returned HTTP 200 consistently after all changes compiled
- No regressions in text or image AI routes (both still 200)

## Design decisions

**Category pills over dropdown**: Pill row is scannable at a glance and a single tap — no open/close interaction. Four categories fit without wrapping on 320px+ screens.

**Amber zone at 75%**: Common nutrition guidance flags "approaching your limit" before hitting it. Amber at 75% gives a visual warning before the user overshoots.

**Hover-reveal delete**: In a list of items, always-visible delete buttons add visual noise and invite accidental taps. Hover-reveal (and always-visible on touch devices via `group-hover`) keeps the list clean while keeping the action accessible.

**No modal confirmations for delete**: Single food entries are low-stakes. An "are you sure?" modal is friction without proportionate benefit. The entry reappears if fetched again within the same session and can be re-added.
